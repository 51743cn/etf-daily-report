# ETF 每日自动分析报告 — 一次性配置指南

> 配置完成后，每个交易日 **15:35（北京时间）**自动运行，分析结果发送至你的 Gmail 邮箱。
> 完全免费，无需本地电脑保持开机。

---

## 第一步：获取 Gmail 应用专用密码（约3分钟）

> Gmail 不允许直接用登录密码发邮件，需要生成一个专用的16位密码。

1. 打开浏览器，登录你的 Google 账号
2. 进入：**https://myaccount.google.com/security**
3. 找到「两步验证」，如果还没开启，先开启它（手机验证即可）
4. 开启后，进入：**https://myaccount.google.com/apppasswords**
5. 在「选择应用」中选 **邮件**，「选择设备」中选 **Windows 计算机**
6. 点击「生成」，会看到一个 **16位字母密码**（例如：`abcd efgh ijkl mnop`）
7. **复制这个密码，去掉空格保存好**（只显示一次！）

---

## 第二步：创建 GitHub 仓库（约3分钟）

> GitHub 是一个免费的代码托管平台，我们用它来运行自动化任务。

1. 打开 **https://github.com** 注册或登录账号
2. 点击右上角的 **「+」→「New repository」**
3. Repository name 填：`etf-daily-report`
4. 选 **Public**（公开，这样免费用时不限）
5. 勾选「Add a README file」
6. 点击「Create repository」

---

## 第三步：上传三个文件（约5分钟）

在刚创建的仓库页面：

### 上传 `daily_analysis.py`

1. 点击「Add file」→「Upload files」
2. 把 `daily_analysis.py` 拖进去
3. 点击「Commit changes」

### 上传 `requirements.txt`

1. 再次点击「Add file」→「Upload files」
2. 把 `requirements.txt` 拖进去
3. 点击「Commit changes」

### 创建 `.github/workflows/daily_report.yml`

这个文件需要手动建文件夹路径：

1. 点击「Add file」→「Create new file」
2. 在文件名框输入：`.github/workflows/daily_report.yml`
   （输入 `/` 会自动创建文件夹）
3. 把 `daily_report.yml` 的全部内容粘贴进去
4. 点击「Commit new file」

---

## 第四步：填入你的邮箱信息（约2分钟）

> 这是最重要的一步，邮箱密码要安全存放在 GitHub Secrets 里，不会泄露。

1. 在仓库页面，点击顶部的 **「Settings」**
2. 左侧菜单找到「**Secrets and variables**」→「**Actions**」
3. 点击「**New repository secret**」，依次添加以下3个：

| Secret 名称       | 填入内容                          |
|------------------|----------------------------------|
| `GMAIL_SENDER`   | 你的 Gmail 地址（如 abc@gmail.com） |
| `GMAIL_APP_PASS` | 第一步获取的16位密码（去掉空格）    |
| `GMAIL_RECEIVER` | 收件邮箱地址（可以和发件一样）      |

每次填完点「Add secret」。

---

## 第五步：手动测试一次（验证配置是否正确）

1. 在仓库页面点击顶部「**Actions**」标签
2. 左侧找到「ETF每日收盘分析报告」
3. 右侧点击「**Run workflow**」→「Run workflow」（绿色按钮）
4. 等待约 2-3 分钟，查看运行结果
5. 如果显示绿色 ✅ → 检查你的邮箱，应该已收到报告
6. 如果显示红色 ❌ → 点进去看错误信息，99% 是邮箱密码填错了

## 常见问题

**Q: 公开仓库会不会泄露我的邮箱密码？**
A: 不会。密码存在 Secrets 里，任何人都看不到，包括你自己的仓库访问者。

**Q: 周末和节假日会运行吗？**
A: 周末不运行（已设置只在周一至周五）。A股节假日 GitHub 不知道，但即使运行，抓到的数据是最后一个交易日的，不影响。

**Q: 60天不操作 GitHub 会停止吗？**
A: GitHub Actions 在仓库60天无 commit 会暂停。解决办法：偶尔去仓库点一下「Run workflow」手动触发，或者每隔两个月随便修改一个文件。

**Q: 如何停止自动运行？**
A: 进入仓库「Actions」→ 点击左侧工作流名称 → 右上角「...」→「Disable workflow」即可暂停。
